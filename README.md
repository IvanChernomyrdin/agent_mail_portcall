# Notify Listener Agent — агент для пересылки уведомлений из PostgreSQL в Redpanda

## Назначение

Агент слушает уведомления PostgreSQL через механизм LISTEN/NOTIFY. При получении уведомления с ID записи загружает данные из БД и отправляет их в Redpanda для дальнейшей обработки.

---

## Архитектура и ключевые решения

- **LISTEN/NOTIFY** — подписка на события PostgreSQL
- **Загрузка данных** — по полученному ID из БД извлекаются данные для отправки
- **Отправка в Redpanda** — данные публикуются в топик для последующей обработки (например, отправка email)
- **Восстановление после падения** — при запуске агент проверяет необработанные записи и отправляет их в Redpanda

---

## Результат

- Асинхронная передача данных из PostgreSQL в Redpanda
- Отвязка отписывающей логики от базы данных
- Надёжное восстановление после перезапусков
- Низкая задержка за счёт push-модели уведомлений
