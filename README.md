# Notify Service — асинхронная отправка уведомлений на почту

## Назначение

Микросервис для асинхронной отправки email-уведомлений через событийную архитектуру. Состоит из двух обработчиков: первый слушает уведомления PostgreSQL (LISTEN/NOTIFY), второй — обрабатывает сообщения из Redpanda (Kafka-совместимый брокер). Обеспечивает надежную доставку, восстановление после падений и параллельную обработку.

---

## Архитектура и ключевые решения

### Обработчик 1 — Listener (PostgreSQL → Redpanda)
- Слушает уведомления PostgreSQL через механизм `LISTEN/NOTIFY`
- При получении уведомления с ID записи:
  - Загружает данные из БД для отправки на почту
  - Обновляет статус записи
  - Формирует JSON и отправляет в Redpanda

### Обработчик 2 — Worker (Redpanda → Email)
- Читает сообщения из Redpanda (1 consumer)
- Обрабатывает их **10 воркерами параллельно**
- Формирует и отправляет email через SMTP

### Восстановление после падения
- При запуске сервис загружает все необработанные записи из БД
- Кладёт их в слайс и отправляет в Redpanda, если они подходят по статусу
- Гарантирует, что ни одно уведомление не потеряно

---

## Результат

- Полностью асинхронная отправка почты через event-события
- Разделение ответственности: Listener — подготовка данных, Worker — отправка
- Параллельная обработка (10 воркеров) для повышения пропускной способности
- Надёжное восстановление после падений сервиса
- Использование Redpanda как высокопроизводительного брокера сообщений
